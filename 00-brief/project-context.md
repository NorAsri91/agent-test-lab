# Project Context

## Purpose

This lab is a controlled sandbox for testing and comparing AI coding agents before using them on real production projects. Each test folder covers a different phase of the software development lifecycle.

## Sample Project: MCTicket

**MCTicket** is a SaaS ticketing and job-sheet system built for computer repair shops.

Core features envisioned:
- Create and manage repair tickets / job sheets
- Track device intake, diagnostics, repair status, and delivery
- Customer lookup and communication history
- Technician assignment and workload view
- Parts inventory tracking per job

## Constraints

- **No database** — all data is mocked or local.
- **No production credentials** — no `.env` files with real secrets.
- **No deployment** — local-only development and testing.
- **Isolated environment** — changes here do not affect any real project.
